# 🔍 Journal Entry Control System (분개장 이상징후 탐지기)

> **회계감사 실무(ICFR) 관점의 분개장 오류통제 절차를 데이터 기반으로 자동화한 Python & Streamlit 프로토타입 프로젝트입니다.**
> 본 프로젝트는 수만 건의 전표를 전수 조사하여 리스크가 높은 이상징후 전표를 신속하게 스크리닝함으로써, 주니어 감사인의 반복 리소스를 줄이고 리스크 중심 감사를 수행할 수 있도록 설계되었습니다.

---

## ✨ 핵심 구현 기능 (3대 통제 로직)

### 1. 우회 입력 통제 (Outside Hours & Weekend Control)
* **목적**: 정상적인 영업시간 외에 입력된 전표를 탐지하여 경영진의 우회 입력 및 권한 남용 위험 식별
* **로직**: `POSTING_DATE`가 주말(토/일)이거나 `INPUT_TIME`이 야간 시간대(20:00 ~ 익일 06:00)인 전표 스크리닝

### 2. 부정 위험 통제 (Fraud Risk Keyword Screening)
* **목적**: 비정상적인 자금 유출이나 부적절한 거래 징후를 적요란 키워드 기반으로 스크리닝
* **로직**: `DESCRIPTION` 내 위험 키워드(*임원 대여, 손실 보전, 현금 인출, 정산 보류, 대표이사 개인, 자문료 증빙 미비*) 포함 여부 분석
* **특징**: `st.multiselect` 및 사용자 정의 텍스트 입력을 통한 실시간 키워드 커스텀 필터링 기능 제공

### 3. 데이터 정합성 통제 (Benford's Law Analysis)
* **목적**: 인위적인 숫자 조작 및 가공 전표 삽입 시 나타나는 통계적 이상치 분석
* **로직**: 차변(`DEBIT`) 금액의 첫째 자리 숫자 분포를 계산하여 벤포드의 법칙 이론 곡선과 비교
* **시각화**: `st.line_chart`를 통한 이론 분포 vs 실제 분포 실시간 비교 및 오차율(8.0%p 이상) 발생 시 경고 알림

## 📊 Mock Data 테스트 및 검증 결과

정확한 로직 검증을 위해 의도적으로 이상징후를 심어둔 100건의 가상 분개장 데이터(Mock Data)를 활용해 렌더링 검증을 완료했습니다.

* **전체 전표 건수**: 100건
* **우회 입력 탐지**: 15건 (주말 8건, 야간 7건)
* **위험 키워드 탐지**: 12건
* **🚨 고위험군 (우회 입력 + 위험 키워드 중복)**: **6건**
* **📈 벤포드 분석 결과**: 소모품비/지급수수료 계정의 금액 조작 시뮬레이션으로 인해 **숫자 '7'에서 이상치 경고 발생**

> 💡 **인사이트**: 단일 필터링만 적용 시 탐지율이 높아 감사인의 업무 과중(False Positive)을 유발할 수 있으나, **[우회 입력 + 위험 키워드 + 벤포드 통계 이상치]를 결합한 스코어링 모델**을 통해 최종 6건의 '최고 위험 전표'를 압축하여 리스크 중심 감사의 실효성을 증명했습니다.
